## ✍️ 手动整理

### JSON 格式模板

```json
[
  {
    "category": "KET",
    "part": "Reading Part 1",
    "question_type": "single_choice",
    "content": "What does this sign say? [图片：NO SMOKING]",
    "options": [
      {"key": "A", "text": "You can smoke here."},
      {"key": "B", "text": "You cannot smoke here."},
      {"key": "C", "text": "Smoking area"}
    ],
    "answer": "B",
    "explanation": "NO SMOKING 意思是\"禁止吸烟\"。",
    "difficulty": 1,
    "source": "2023 年真题",
    "tags": ["标志识别", "词汇"]
  }
]
```

### 字段说明

| 字段 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `category` | ✅ | KET 或 PET | `"KET"` |
| `part` | ✅ | 题目部分 | `"Reading Part 1"` |
| `question_type` | ✅ | 题型 | `"single_choice"` |
| `content` | ✅ | 题目内容 | `"What does this sign say?"` |
| `options` | ✅ | 选项数组 | `[{"key":"A","text":"..."}]` |
| `answer` | ✅ | 正确答案 | `"B"` |
| `explanation` | ❌ | 答案解析 | `"NO SMOKING 意思是..."` |
| `difficulty` | ❌ | 难度 1-5 | `1` |
| `source` | ❌ | 题目来源 | `"2023 年真题"` |
| `tags` | ❌ | 标签数组 | `["词汇", "语法"]` |

### 题型列表

- `single_choice`: 单选题
- `multiple_choice`: 多选题
- `fill_blank`: 填空题
- `matching`: 匹配题
- `word_formation`: 词形转换

## 📥 导入系统

### 方法一：命令行导入（推荐）

适合批量导入大量题目。

```bash
# 1. 进入后端目录
cd /home/www/xxb-ket-pet/backend

# 2. 安装依赖（首次使用）
npm install

# 3. 执行导入
npm run import-questions -- --file=../scripts/collected-questions-20260417.json

# 4. 查看结果
# 导入完成后显示成功/失败数量
```

### 方法二：API 上传

适合通过网页界面上传。

```bash
# 使用 curl
curl -X POST http://localhost:3001/api/questions/import \
  -F "file=@questions.json"
```

### 方法三：SQL 直接导入

适合超大批量（1000+ 题目）。

```bash
# 1. 准备 SQL 文件
psql -h localhost -U xxb -d xxb_db -f import-questions.sql

# 2. 查看导入结果
docker exec -it xxb-postgres psql -U xxb -d xxb_db -c "SELECT COUNT(*) FROM questions"
```

## 📊 题库管理

### 查看题目

```bash
# API 获取题目列表
curl http://localhost:3001/api/questions/list

# 按类别过滤
curl http://localhost:3001/api/questions/list?category=KET

# 分页
curl http://localhost:3001/api/questions/list?page=1&limit=20
```

### 统计信息

```bash
# 获取题目统计
curl http://localhost:3001/api/questions/stats/summary
```

**响应示例**:
```json
{
  "stats": [
    {"category": "KET", "total": 50, "active": 48, "avg_difficulty": 1.8},
    {"category": "PET", "total": 30, "active": 28, "avg_difficulty": 2.5}
  ]
}
```

### 更新题目

```bash
# PUT /api/questions/:id
curl -X PUT http://localhost:3001/api/questions/1 \
  -H "Content-Type: application/json" \
  -d '{"difficulty": 2, "explanation": "新的解析"}'
```

### 删除题目

```bash
# DELETE /api/questions/:id
curl -X DELETE http://localhost:3001/api/questions/1
```

## 🔧 常见问题

### Q: 导入失败怎么办？

A: 检查以下几点：
1. JSON 格式是否正确
2. 必需字段是否完整
3. category 是否为 KET 或 PET
4. 查看错误日志获取详细信息

### Q: 如何批量更新题目？

A: 使用 SQL 直接更新：
```sql
UPDATE questions SET difficulty = 2 WHERE category = 'KET';
```

### Q: 如何导出题库？

A: 使用 API 或 SQL：
```bash
# API
curl http://localhost:3001/api/questions/list?limit=1000

# SQL
docker exec -it xxb-postgres pg_dump -U xxb xxb_db questions > backup.sql
```

### Q: 题目有重复怎么办？

A: 导入工具会自动去重，也可以手动删除：
```sql
DELETE FROM questions WHERE content IN (
  SELECT content FROM questions GROUP BY content HAVING COUNT(*) > 1
);
```
